Firebase Basics

Getting Started with Firebase

## What is Firebase?

Firebase is a platform developed by Google that helps you build apps quickly. Think of it as a complete backend solution - you don't need to worry about servers, databases, or complex infrastructure.

## Why Use Firebase?

- **No Server Setup**: Everything runs in the cloud
- **Real-time Updates**: Changes sync instantly across all devices
- **Built-in Authentication**: Easy user login systems
- **File Storage**: Store images, videos, and files easily
- **Free to Start**: Generous free tier for learning

## What You'll Need

1. A Google account (Gmail)
2. Basic HTML and JavaScript knowledge
3. A text editor (like VS Code)

## Creating Your First Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click "Add Project"
3. Give your project a name
4. Choose your settings (you can disable analytics for now)
5. Click "Create Project"

Understanding Firestore Database

## What is Firestore?

Firestore is Firebase's database. Unlike traditional databases with tables and rows, Firestore uses:

- **Collections**: Like folders that hold documents
- **Documents**: Individual items with data (like a file)

### Structure Example

## Adding Documents to Firestore

There are two ways to add data:

### Method 1: Auto-generated ID
Firestore creates a unique ID for you.

### Method 2: Custom ID

## When to Use Each Method

- **Auto ID**: Use for most cases (posts, comments, messages)
- **Custom ID**: Use when you need a specific ID (user profiles using their email)

Reading Data (GET)

## Getting Data from Firestore

There are several ways to read data:

### 1. Get All Documents
Fetch all documents in a collection.

### 2. Get One Document
Fetch a specific document by its ID.

### 3. Real-time Updates
Automatically update when data changes (like live updates!).

## Understanding .then()

Firebase operations are asynchronous. Think of `.then()` as "when the data arrives, do this".
Get All Posts - javascript
// Fetch all posts
db.collection('posts').get()
.then((querySnapshot) => {
    querySnapshot.forEach((doc) => {
        console.log('Post ID:', doc.id);
        console.log('Data:', doc.data());
        console.log('Title:', doc.data().title);
        console.log('---');
    });
})
.catch((error) => {
    console.error('Error:', error);
});
Get Single Document - javascript
// Get specific post by ID
const postId = 'abc123';

db.collection('posts').doc(postId).get()
.then((doc) => {
    if (doc.exists) {
        console.log('Post data:', doc.data());
        console.log('Title:', doc.data().title);
    } else {
        console.log('Post not found!');
    }
})
.catch((error) => {
    console.error('Error:', error);
});
Display Posts in HTML - html
<div id="postsList"></div>

// Fetch and display all posts

Updating Data (UPDATE)

## Updating Documents

There are two methods:

### update()
Updates specific fields, leaves others unchanged.

### set()
Replaces the entire document.

## When to Use Which

- **update()**: Change only certain fields (recommended)
- **set()**: Replace everything (careful!)
- **set() with merge**: Updates fields but keeps others
Update Specific Fields - javascript
// Update only the title and content
const postId = 'abc123';

db.collection('posts').doc(postId).update({
    title: 'Updated Title',
    content: 'Updated content here',
    lastModified: new Date()
})
.then(() => {
    console.log('Post updated successfully!');
})
.catch((error) => {
    console.error('Error updating:', error);
});

Deleting Data (DELETE)

## Deleting Documents

You can delete:
- Entire documents
- Specific fields within documents

## Important Notes

‚ö†Ô∏è **Deletion is permanent!** There's no undo.

‚ö†Ô∏è **Deleting a document doesn't delete its subcollections.**

## Best Practices

1. Always confirm before deleting
2. Consider soft deletes (marking as deleted) instead
3. Make sure you have the correct document ID
Delete a Document - javascript

function deletePost(postId) {
    // Ask for confirmation
    const confirmed = confirm('Are you sure you want to delete this post?');
    
    if (confirmed) {
        db.collection('posts').doc(postId).delete()
        .then(() => {
            // Remove from page

Filtering & Sorting Data

## Querying Data

Firestore lets you filter and sort data before fetching it.

## Common Queries

- **where()**: Filter by field value
- **orderBy()**: Sort results
- **limit()**: Get only X number of results

## Query Operators

- `==` Equal to
- `!=` Not equal
- `>` Greater than
- `>=` Greater than or equal
- `<` Less than
- `<=` Less than or equal
- `in` Value in array
- `array-contains` Array contains value
Filter Posts by Author - javascript
// Get only posts by specific author
db.collection('posts')
  .where('author', '==', 'John Doe')
  .get()
  .then((querySnapshot) => {
      querySnapshot.forEach((doc) => {
          console.log('Post:', doc.data().title);
      });
  });
Sort and Limit Results - javascript
// Get latest 10 posts
db.collection('posts')
  .orderBy('createdAt', 'desc')  // newest first
  .limit(10)
  .get()
  .then((querySnapshot) => {
      console.log('Latest 10 posts:');
      querySnapshot.forEach((doc) => {
          const post = doc.data();
          console.log(post.title, post.createdAt);
      });
  });
Multiple Filters - javascript
// Get posts by author John, created in 2024, sorted by likes
db.collection('posts')
  .where('author', '==', 'John Doe')
  .where('year', '==', 2024)
  .orderBy('likes', 'desc')
  .get()
  .then((querySnapshot) => {
      querySnapshot.forEach((doc) => {
          console.log(doc.data());
      });
  });
Search with Greater/Less Than - javascript
// Get products priced between $10 and $50
db.collection('products')
  .where('price', '>=', 10)
  .where('price', '<=', 50)
  .orderBy('price', 'asc')
  .get()
  .then((querySnapshot) => {
      console.log('Affordable products:');
      querySnapshot.forEach((doc) => {
          const product = doc.data();
          console.log(product.name, '$' + product.price);
      });
  });

Understanding Local Storage

## What is Local Storage?

Local Storage is like a mini database in your browser. It stores data even after you close the page!

## Local Storage vs Firestore

**Local Storage:**
- ‚úÖ Works offline
- ‚úÖ No setup needed
- ‚úÖ Instant (no internet)
- ‚ùå Only on that device
- ‚ùå Limited to ~5MB
- ‚ùå Not secure

**Firestore:**
- ‚úÖ Works on all devices
- ‚úÖ Unlimited storage
- ‚úÖ Secure
- ‚úÖ Real-time sync
- ‚ùå Needs internet
- ‚ùå Requires setup

## When to Use Local Storage

- User preferences (theme, language)
- Temporary form data
- Shopping cart (before checkout)
- App settings
- Game scores (local only)

## When to Use Firestore

- User accounts
- Social media posts
- Any data that needs to sync
- Data to share between users

    const isDark = document.body.style.background === 'rgb(51, 51, 51)';
